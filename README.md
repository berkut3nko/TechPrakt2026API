# TechPrakt 2026: PHP Frameworks API

Цей репозиторій містить виконання практичного завдання з вивчення сучасних PHP-фреймворків. У рамках проєкту реалізовано REST API для сутності Project (CRUD операції) з використанням двох різних архітектурних підходів: Laravel (Active Record) та Symfony (Data Mapper).

## 📌 Залежності та Системні Вимоги (Prerequisites)

Перед початком роботи переконайтеся, що на вашому комп'ютері встановлено наступне програмне забезпечення:

PHP: Версія 8.2 або вище.

Composer: Версія 2.x для управління залежностями.

СУБД: MySQL 8.0+ або PostgreSQL 15+.

Клієнт API: Postman (для тестування ендпоінтів).

CLI (Опціонально): Symfony CLI для зручного запуску сервера Symfony.

## 🚀 Гайд із запуску (Laravel)

### Перейдіть до директорії Laravel:

```bash
cd laravel
```


### Налаштування оточення:
Скопіюйте файл конфігурації (якщо він ще не створений) та згенеруйте ключ:
```bash
cp .env.example .env
php artisan key:generate
```


### Конфігурація Бази Даних:
Відкрийте файл .env та вкажіть ваші дані доступу до локальної бази даних (змініть DB_DATABASE, DB_USERNAME, DB_PASSWORD).

### Запуск міграцій:
Створіть таблиці в базі даних:
```bash
php artisan migrate
```

### Запуск локального сервера:
```bash
php artisan serve
```

API буде доступне за адресою: http://127.0.0.1:8000/api/projects

## 🚀 Гайд із запуску (Symfony)

Відкрийте нову вкладку термінала та перейдіть до директорії Symfony:
```bash
cd symfony
```

### Конфігурація Бази Даних:
Відкрийте файл .env (або .env.local) та відредагуйте змінну DATABASE_URL.
Приклад для MySQL:
DATABASE_URL="mysql://root:password@127.0.0.1:3306/symfony_db?serverVersion=8.0.32&charset=utf8mb4"

### Створення БД та запуск міграцій:
Виконайте наступні команди через Doctrine:
```bash
php bin/console doctrine:database:create
php bin/console doctrine:migrations:migrate
```

### Запуск локального сервера:
Якщо встановлено Symfony CLI:
```bash
symfony server:start
```

Або через вбудований PHP сервер:
```bash
php -S 127.0.0.1:8001 -t public
```

(Зверніть увагу на порт 8001, щоб уникнути конфлікту з Laravel).

## 📖 Імпорт документації в Postman

В корені репозиторію знаходяться два JSON-файли з готовими Postman-колекціями, які містять детальну документацію, тестові скрипти та приклади відповідей сервера:
```text
laravel_project_api.json

symfony_project_api.json
```

Як імпортувати:

1. Відкрийте програму Postman.

2. У верхньому лівому куті натисніть на кнопку "Import".

3. У вікні, що з'явилося, виберіть вкладку "File" (або просто перетягніть файли у вікно).

4. Виберіть обидва файли .json з вашої папки frameworks.

5. Натисніть "Import".

### Як читати документацію в Postman:

Після імпорту, виберіть колекцію на лівій панелі. У центральному вікні натисніть на іконку документації (справа) або на синє посилання View complete documentation, щоб побачити повний архітектурний опис, таблиці бази даних та правила валідації.
